# MGT-030: ScalarResult `.yield_per()` / `.partitions()` streaming

## Description

Add batched iteration support to `ScalarResult` for processing large result
sets without loading everything into memory.

- `.yield_per(size)` — configure the cursor to fetch `size` documents at a time
  from the server (maps to MongoDB's `batch_size`).
- `.partitions(size)` — yield results in chunks of `size` as lists, useful for
  batch processing.

## Expected API

```python
# Stream results in batches of 100
result = session.scalars(select(User)).yield_per(100)
for user in result:
    process(user)

# Process in chunks
for batch in session.scalars(select(User)).partitions(500):
    bulk_process(batch)  # batch is List[User] of up to 500
```

## Implementation Notes

- `.yield_per(size)` maps to pymongo's `cursor.batch_size(size)`.
- `.partitions(size)` is a Python-level chunking over the cursor.
- Both require `ScalarResult.__iter__` (MGT-020) to be implemented first.
- Consider cursor timeout settings for long-running iterations.
