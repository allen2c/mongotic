# MGT-029: `session.expunge()` / `session.expire()`

## Description

Add fine-grained session object management:

- `session.expunge(instance)` — detach the instance from the session. Further
  attribute changes will not be tracked.
- `session.expire(instance)` — mark the instance as stale. The next attribute
  access should trigger a reload from the database (lazy refresh).

## Implementation Notes

- `expunge()` should remove the instance from all staging lists and clear
  `instance._session`.
- `expire()` requires a descriptor-level hook or proxy to intercept attribute
  access and trigger `refresh()` — this adds complexity.
- Consider whether `expire()` is worth the complexity without an identity map.
  A simpler approach: `expire()` just clears local field values and sets a
  flag, requiring an explicit `refresh()` call.
