# MGT-022: String operators — `.like()`, `.contains()`, `.startswith()`, `.endswith()`

## Description

Add SA v2-style string matching operators to `ModelField`, mapped to MongoDB
`$regex` queries.

## Expected API

```python
# SQL LIKE equivalent (% is wildcard)
stmt = select(User).where(User.name.like("Al%"))

# Case-insensitive LIKE
stmt = select(User).where(User.name.ilike("al%"))

# Convenience methods
stmt = select(User).where(User.email.contains("@gmail.com"))
stmt = select(User).where(User.name.startswith("Al"))
stmt = select(User).where(User.name.endswith("son"))
```

## Implementation Guidelines

1. **Add new operators to `Operator` enum**: `REGEX` (or reuse a generic
   approach with `$regex`).
2. **Add methods to `ModelField`**:
   - `like(self, pattern: str)` — convert SQL `%` wildcard to regex `.*`,
     `_` to `.`, and wrap with `^...$`. Return `ModelFieldOperation` with
     `$regex` value.
   - `ilike(self, pattern: str)` — same as `like()` but with `$options: "i"`.
   - `contains(self, value: str)` — `$regex: re.escape(value)` (substring match).
   - `startswith(self, value: str)` — `$regex: ^re.escape(value)`.
   - `endswith(self, value: str)` — `$regex: re.escape(value)$`.
3. **Update `to_mongo_filter()`** to emit `{"field": {"$regex": pattern, "$options": opts}}`
   for regex-type operations.
4. **Escape user input** with `re.escape()` in `contains()`, `startswith()`,
   `endswith()` to prevent regex injection. `like()` / `ilike()` should only
   convert `%` and `_` wildcards, escaping everything else.

### Key design considerations

- Always use `re.escape()` on literal parts to prevent regex injection.
- `like()` follows SQL convention: `%` = any chars, `_` = single char.
- `ilike()` is case-insensitive variant (common SA pattern).
- MongoDB regex queries cannot use indexes unless the pattern is anchored
  with `^` — document this performance consideration.

## Acceptance Criteria

- [x] `.like("Al%")` produces `{"$regex": "^Al.*$"}` filter
- [x] `.ilike("al%")` adds `{"$options": "i"}`
- [x] `.contains("gmail")` matches substrings
- [x] `.startswith("Al")` matches prefix
- [x] `.endswith("son")` matches suffix
- [x] User input is escaped to prevent regex injection
