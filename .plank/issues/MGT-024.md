# MGT-024: `Select.distinct()` support

## Description

Add `.distinct(field)` to the query API, mapping to MongoDB's
`collection.distinct()`. This returns unique values for a given field,
optionally filtered by `.where()` conditions.

## Expected API

```python
# Get all unique roles
roles = session.scalars(select(User).distinct(User.role))
# → ["admin", "member", "guest"]

# Distinct with filter
active_roles = session.scalars(
    select(User).where(User.active == True).distinct(User.role)
)
```

## Implementation Guidelines

1. **Add `distinct(self, field: ModelField)` to `Select`** that stores the
   distinct field. This marks the query as a distinct query.
2. **Update `ScalarResult`** to detect when a `Select` has a `distinct` field
   set and use `collection.distinct(field_name, filter)` instead of
   `collection.find()`.
3. **Return type**: `collection.distinct()` returns a plain list of values
   (not documents). The `ScalarResult` should return these values directly
   rather than hydrating into model instances.
4. **Consider a separate result type** or make `ScalarResult` generic enough
   to handle both document results and scalar value lists.

### Key design considerations

- SA v2's `distinct()` is on the `Select` statement and deduplicates entire
  rows. MongoDB's `distinct()` is per-field. Align with MongoDB's native
  semantics but use SA-like syntax.
- `.distinct()` with `.order_by()` is not supported by MongoDB's `distinct()`
  command — document this limitation or raise an error.
- `.limit()` / `.offset()` also don't apply to `distinct()` — handle gracefully.
- Type safety: `select(User).distinct(User.role)` should ideally return
  `List[str]` not `List[User]`, but this is hard to express with current
  generics. Consider returning `List[Any]` for distinct queries and document
  the trade-off.

## Acceptance Criteria

- [x] `select(User).distinct(User.role)` returns unique values for that field
- [x] `.where()` conditions filter before distinct
- [x] Unsupported combos (order_by + distinct) documented in `.distinct()` docstring
- [x] Tests in `tests/test_distinct.py`
