# MGT-005: ScalarResult.one() + one_or_none() + MultipleResultsFound

## Description

Add `.one()` and `.one_or_none()` to `ScalarResult`, and introduce the
`MultipleResultsFound` exception. These are the strict counterparts to
`.first()`.

## API

```python
# .one() — exactly one result required
user = session.scalars(select(User).where(User.email == "a@b.com")).one()
# 0 results  → raises NotFound
# 1 result   → returns User
# 2+ results → raises MultipleResultsFound

# .one_or_none() — zero or one result
user = session.scalars(select(User).where(User.email == "a@b.com")).one_or_none()
# 0 results  → returns None  (does NOT raise)
# 1 result   → returns User
# 2+ results → raises MultipleResultsFound
```

## Difference from .first()

| method | 0 results | 1 result | 2+ results |
|--------|-----------|----------|------------|
| `.first()` | `None` | instance | instance (first one) |
| `.one_or_none()` | `None` | instance | `MultipleResultsFound` |
| `.one()` | `NotFound` | instance | `MultipleResultsFound` |

## Implementation Notes

- Both methods fetch at most 2 documents (`limit=2`) for efficiency
- Add `MultipleResultsFound` to `mongotic/exceptions.py`
- Export `MultipleResultsFound` from `mongotic/__init__.py`

## Acceptance Criteria

- [ ] `MultipleResultsFound` in `exceptions.py` and exported
- [ ] `.one()` raises `NotFound` on 0, returns instance on 1, raises `MultipleResultsFound` on 2+
- [ ] `.one_or_none()` returns `None` on 0, returns instance on 1, raises `MultipleResultsFound` on 2+
- [ ] Tests cover all paths for both methods
