# MGT-031: `ScalarResult.first()` ignores `order_by` / `offset`

## Description

`ScalarResult.first()` currently calls `collection.find_one(filter)` directly,
bypassing `_build_cursor()`. This means any `.order_by()` or `.offset()` applied
to the `Select` statement is silently ignored.

```python
# Expected: most recent user (respects order_by)
# Actual: arbitrary first document in collection order
user = session.scalars(
    select(User).order_by(-User.created_at)
).first()
```

SA v2's `first()` respects `ORDER BY` and `OFFSET`, so mongotic should too.

## Fix

Replace the `find_one` call with `_build_cursor().limit(1)` and take the first
element, matching the behaviour of `one()` and `one_or_none()`.

```python
def first(self) -> Optional[_T]:
    cursor = self._build_cursor().limit(1)
    docs = list(cursor)
    return self._hydrate(docs[0]) if docs else None
```

## Acceptance Criteria

- [x] `first()` respects `.order_by()` from the `Select` statement
- [x] `first()` respects `.offset()` from the `Select` statement
- [x] `first()` still returns `None` when no document matches
