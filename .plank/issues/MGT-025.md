# MGT-025: `session.merge(instance)` — upsert semantics

## Description

SA v2's `session.merge()` copies the state of a given instance onto a
persistent instance with the same primary key. If no such persistent instance
exists, it inserts a new one. This maps naturally to MongoDB's
`update_one(..., upsert=True)`.

## Expected API

```python
with Session() as session:
    # If a user with this _id exists, update it; otherwise insert
    user = User(_id="507f1f77bcf86cd799439011", name="Alice", age=30)
    merged = session.merge(user)
    session.flush()

    # merged is the persisted instance
    print(merged._id)  # "507f1f77bcf86cd799439011"
```

## Implementation Guidelines

1. **Add `merge(self, instance: MongoBaseModel) -> MongoBaseModel` to `_Session`**:
   - If `instance._id` is set:
     - Use `collection.find_one({"_id": ObjectId(instance._id)})` to check existence.
     - If found: update the existing document with `instance`'s current field values,
       track it in `_update_instances`, and return the instance bound to the session.
     - If not found: stage it as a new insert via `_add_instances`.
   - If `instance._id` is not set:
     - Treat as a new insert (same as `session.add()`).
2. **Return value**: the merged instance with `_session` set and `_id` populated.
3. **Add `merge` to the `Session` Protocol**.

### Key design considerations

- SA v2's `merge()` has complex identity-map semantics (detached vs persistent).
  Since mongotic has no identity map, simplify: `merge()` = "upsert this
  document". This is a pragmatic MongoDB-first interpretation.
- At `flush()` time, the actual `update_one(filter, {"$set": ...}, upsert=True)`
  call should be used for merged instances with `_id`.
- Consider adding a `_merge_instances` staging list (separate from `_add` and
  `_update`) so `flush()` can handle them with `upsert=True`.
- Document that `merge()` is **eager on flush** — the upsert happens when
  `flush()` / `commit()` is called, not immediately.

## Acceptance Criteria

- [x] `session.merge(instance)` with existing `_id` updates the document on flush
- [x] `session.merge(instance)` with non-existing `_id` inserts on flush
- [x] `session.merge(instance)` with no `_id` behaves like `session.add()`
- [x] Returns the merged instance with `_session` bound
- [x] Added to `Session` Protocol
- [x] Tests in `tests/test_session_merge.py`
