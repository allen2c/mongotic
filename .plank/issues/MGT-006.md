# MGT-006: Bulk update via session.execute()

## Description

Introduce an `Update` statement and a `update()` factory function so bulk
updates can be expressed as first-class statements passed to `session.execute()`.

## Expected API

```python
from mongotic import update

stmt = update(User).where(User.role == "guest").values(role="member")
affected = session.execute(stmt)  # returns int (modified count)
```

## Implementation Notes

- `update(Model)` → `Update(orm_model=Model)`
- `Update` has `.where()` (same as `Select`) and `.values(**kwargs)`
- `session.execute(Update)` calls `collection.update_many(filter, {"$set": values})`
- Executes immediately (not staged via `add`/`commit`)
- Returns `int` — number of modified documents

## Acceptance Criteria

- [x] `update()` importable from `mongotic`
- [x] `.where()` and `.values()` chainable on `Update`
- [x] `session.execute(update_stmt)` runs `update_many` and returns count
- [x] Tests added to `tests/test_orm.py`
