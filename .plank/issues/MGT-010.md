# MGT-010: Introduce ScalarResult class

## Description

Introduce a `ScalarResult` class that wraps the pymongo cursor after execution
and provides a SQLAlchemy v2-style result interface. This is the object returned
by `session.scalars(stmt)`.

## Expected API

```python
result = session.scalars(select(User).where(User.age > 18))

result.all()     # List[User]
result.first()   # User | None  (no exception if empty)
result.one()     # User         (raises NotFound or MultipleResultsFound)
result.count()   # int
result.exists()  # bool
```

## Implementation Notes

- `ScalarResult` is constructed by `Session.scalars()`, receives the hydrated
  documents (or a lazy cursor) and the model class
- `.first()` returns `None` (not raises) when no result — differs from v0.2.0
  `QuerySet.first()` which raised `NotFound`
- `.one()` raises `NotFound` (0 results) or `MultipleResultsFound` (2+ results)
- `.count()` uses `count_documents(filter)` — does not load documents
- `.exists()` uses `count_documents(filter, limit=1) > 0`
- `ScalarResult` exported from `mongotic.orm`

## Acceptance Criteria

- [x] `ScalarResult.all()` returns `List[Model]`
- [x] `ScalarResult.first()` returns `Model | None`
- [x] `ScalarResult.one()` raises properly
- [x] `ScalarResult.count()` returns `int`
- [x] `ScalarResult.exists()` returns `bool`
- [x] Tests cover all five methods
