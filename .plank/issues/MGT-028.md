# MGT-028: Column projection — `select(User.name, User.email)`

## Description

Allow `select()` to accept individual `ModelField` arguments instead of the
full model class. This maps to MongoDB's projection feature, returning only
specified fields to reduce data transfer.

## Expected API

```python
# Returns tuples or named tuples, not full model instances
result = session.execute(select(User.name, User.email))
for name, email in result:
    print(name, email)

# Full model (existing behaviour unchanged)
users = session.scalars(select(User)).all()
```

## Implementation Notes

- Requires a new return type (tuples or `Row` objects) separate from model instances.
- `select(ModelField, ...)` should produce `{"projection": {"name": 1, "email": 1}}`.
- Need to distinguish between `select(Model)` and `select(Model.field, ...)` at
  the type level.
- Consider SA v2's `Row` / `Result` vs `ScalarResult` distinction.
