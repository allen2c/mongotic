# MGT-015: Fix KeyError in MongoBaseModelMeta.__getattr__ when class has no annotations

## Description

`MongoBaseModelMeta.__getattr__` directly indexes `cls.__dict__["__annotations__"]`
without guarding against a missing key. If a `MongoBaseModel` subclass defines no
fields of its own (e.g. an abstract base or a mixin with no annotations), the key
`"__annotations__"` will not be present in `cls.__dict__`, causing a `KeyError`
instead of the expected `AttributeError`.

```python
class EmptyModel(MongoBaseModel):
    __databasename__ = "db"
    __tablename__ = "empty"

EmptyModel.nonexistent  # raises KeyError, should raise AttributeError
```

Relevant code (`mongotic/model.py:163`):

```python
if item in cls.__dict__["__annotations__"]:   # ← KeyError if no annotations
    return ModelField(field_name=item, model_class=cls)
else:
    raise e
```

## Fix

Replace the direct dict access with `.get()`:

```python
if item in cls.__dict__.get("__annotations__", {}):
    return ModelField(field_name=item, model_class=cls)
else:
    raise e
```

## Acceptance Criteria

- [ ] Accessing a non-existent attribute on a `MongoBaseModel` subclass with no
      direct annotations raises `AttributeError`, not `KeyError`
- [ ] Existing tests continue to pass
- [ ] `ModelField` descriptor access on annotated fields is unaffected
