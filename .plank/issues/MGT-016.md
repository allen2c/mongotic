# MGT-016: Fix incorrect type annotations on create_engine() parameters

## Description

`create_engine()` in `mongotic/__init__.py` annotates two parameters as
`Optional[Text]` when they are not string types in pymongo's API:

| Parameter | Current annotation | Correct type |
|---|---|---|
| `document_class` | `Optional[Text]` | `Optional[Type[MutableMapping]]` |
| `type_registry` | `Optional[Text]` | `Optional[TypeRegistry]` |

This misleads IDEs and type checkers into accepting `str` values for these
parameters, masking type errors at the call site.

## Fix

Update `mongotic/__init__.py`:

```python
from typing import Any, MutableMapping, Optional, Text, Type
from bson.codec_options import TypeRegistry
from pymongo import MongoClient

def create_engine(
    host: Optional[Text] = None,
    port: Optional[int] = None,
    document_class: Optional[Type[MutableMapping]] = None,
    tz_aware: Optional[bool] = None,
    connect: Optional[bool] = None,
    type_registry: Optional[TypeRegistry] = None,
    **kwargs: Any,
) -> MongoClient:
    ...
```

## Acceptance Criteria

- [ ] `document_class` annotated as `Optional[Type[MutableMapping]]`
- [ ] `type_registry` annotated as `Optional[TypeRegistry]`
- [ ] No runtime behaviour change
- [ ] Existing tests continue to pass
