# MGT-011: Refactor Session to SA v2 interface

## Description

Refactor `Session` (and `sessionmaker`) to expose the SQLAlchemy v2 interface.
Remove `session.query()`. Add `scalars()`, `execute()`, `get()`, `add_all()`,
`flush()`, and `rollback()`.

## Breaking Changes

- `session.query(Model)` **removed**
- `session.scalars(stmt)` replaces it

## New Session Interface

```python
class Session:
    # Querying
    def scalars(self, stmt: Select) -> ScalarResult: ...
    def execute(self, stmt: Update | Delete) -> int: ...  # returns affected count
    def get(self, model: Type[Model], id: str) -> Model | None: ...

    # Writing
    def add(self, instance: Model) -> None: ...
    def add_all(self, instances: List[Model]) -> None: ...  # NEW
    def delete(self, instance: Model) -> None: ...

    # Transaction control
    def flush(self) -> None: ...     # write staged ops, keep transaction open
    def commit(self) -> None: ...    # flush + commit
    def rollback(self) -> None: ...  # abort transaction
    def close(self) -> None: ...     # end pymongo client session

    # Context manager
    def __enter__(self) -> Session: ...
    def __exit__(self, ...): ...     # calls close()
```

## Implementation Notes

- `scalars(stmt)` builds mongo filter from `stmt._filters`, applies sort/limit/offset,
  executes `find()`, returns a `ScalarResult` bound to (collection, stmt, session)
- `execute(stmt)` dispatches on stmt type: `Update` тЖТ `update_many`, `Delete` тЖТ `delete_many`
- `get(Model, id)` calls `find_one({"_id": ObjectId(id)})`, returns `None` if not found
- `add_all(instances)` calls `self.add(i)` for each тАФ or uses `insert_many` in flush for efficiency
- `flush()` executes staged add/update/delete within the open transaction, clears staging lists
- `commit()` calls `flush()` then commits the MongoDB transaction
- `rollback()` calls `client_session.abort_transaction()` if a transaction is active
- `close()` ends the pymongo client session (called by `__exit__`)

## Acceptance Criteria

- [ ] `session.query()` removed
- [ ] `session.scalars(stmt)` works end-to-end with `Select`
- [ ] `session.get(Model, id)` returns instance or `None`
- [ ] `session.add_all([...])` stages multiple instances
- [ ] `session.flush()` writes without committing
- [ ] `session.rollback()` aborts the active transaction
- [ ] `session.commit()` still works (flush then commit)
- [ ] `session.close()` cleans up the pymongo session
- [ ] All existing ORM tests rewritten for new API
