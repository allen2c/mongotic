# MGT-011: Refactor Session to SA v2 interface

## Description

Refactor `Session` (and `sessionmaker`) to expose the SQLAlchemy v2 interface.
Remove `session.query()`. Add `scalars()`, `execute()`, `get()`, `add_all()`,
`flush()`, and `rollback()`.

## Breaking Changes

- `session.query(Model)` **removed**
- `session.scalars(stmt)` replaces it

## New Session Interface

```python
class Session:
    # Querying
    def scalars(self, stmt: Select) -> ScalarResult: ...
    def execute(self, stmt: Update | Delete) -> int: ...  # returns affected count
    def get(self, model: Type[Model], id: str) -> Model | None: ...

    # Writing
    def add(self, instance: Model) -> None: ...
    def add_all(self, instances: List[Model]) -> None: ...  # NEW
    def delete(self, instance: Model) -> None: ...

    # Write control (no MongoDB transaction; each write is document-atomic)
    def flush(self) -> None: ...     # write staged ops to DB immediately
    def commit(self) -> None: ...    # alias for flush()
    def rollback(self) -> None: ...  # discard staged (not yet flushed) changes
    def close(self) -> None: ...     # discard any un-flushed staged changes

    # Context manager
    def __enter__(self) -> Session: ...
    def __exit__(self, ...): ...     # calls close()
```

## Implementation Notes

- No MongoDB transactions — each document write is individually atomic.
- `scalars(stmt)` builds mongo filter from `stmt._filters`, applies sort/limit/offset,
  executes `find()`, returns a `ScalarResult` bound to (collection, stmt, session)
- `execute(stmt)` dispatches on stmt type: `Update` → `update_many`, `Delete` → `delete_many`
- `get(Model, id)` calls `find_one({"_id": ObjectId(id)})`, returns `None` if not found
- `add_all(instances)` calls `self.add(i)` for each
- `flush()` executes all staged add/update/delete directly, then clears staging lists
- `commit()` is an alias for `flush()` (SA v2 API familiarity)
- `rollback()` discards staged changes only — cannot undo already-flushed writes
- `close()` discards un-flushed staged changes (called by `__exit__`)

- [x] `session.query()` removed

- [x] `session.get(Model, id)` returns instance or `None`

- [x] `session.rollback()` discards staged changes; cannot undo flushed writes
