# MGT-009: Introduce Select statement builder

## Description

Introduce a `Select` class and a `select()` factory function as the primary
query entry point, replacing `session.query()`. A `Select` statement is
session-independent and immutable-by-convention — each chaining method
returns a new (or the same mutated) instance.

## Expected API

```python
from mongotic import select

stmt = select(User)
stmt = stmt.where(User.age > 18, User.active == True)
stmt = stmt.order_by(User.name)       # ascending
stmt = stmt.order_by(-User.created_at) # descending (via __neg__ on ModelField)
stmt = stmt.limit(20)
stmt = stmt.offset(40)
```

## Implementation Notes

- `select(Model)` → `Select(orm_model=Model)`
- `Select` stores: `_model`, `_filters`, `_sort`, `_limit`, `_offset`
- `.where(*ModelFieldOperation)` appends to `_filters`; chainable
- `.limit(n)` / `.offset(n)` set bounds; NO default limit (None = no limit)
- `.order_by(*fields)` — `ModelField` for ascending, add `__neg__` to
  `ModelField` to return a `ModelFieldSort(field, direction=DESC)` token
- Remove `.filter()` / `.filter_by()` (breaking change from v0.2.0)
- `select()` exported from `mongotic/__init__.py`

## Acceptance Criteria

- [ ] `select(Model)` importable from `mongotic`
- [ ] `.where()` accepts `ModelFieldOperation` instances
- [ ] `.order_by()` supports `ModelField` (asc) and `-ModelField` (desc)
- [ ] `.limit()` / `.offset()` work; no default limit
- [ ] `Select` is not bound to any session
- [ ] Old `filter` / `filter_by` removed
