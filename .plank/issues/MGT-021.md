# MGT-021: `session.refresh(instance)`

## Description

SA v2's `session.refresh()` reloads an instance's attributes from the database.
This is useful after bulk updates or when you suspect the in-memory state is
stale. Add this to mongotic's Session.

## Expected API

```python
with Session() as session:
    user = session.scalars(select(User).where(User.name == "Alice")).one()
    print(user.age)  # 25

    # Some external process updates the document...

    session.refresh(user)
    print(user.age)  # 30 (reloaded from DB)
```

## Implementation Guidelines

1. **Add `refresh(self, instance: MongoBaseModel)` to `_Session`**:
   - Read `instance._id` to get the document's `ObjectId`.
   - Raise `ValueError` if `_id` is not set (instance was never persisted).
   - Query `collection.find_one({"_id": ObjectId(instance._id)})`.
   - If the document is not found, raise `NotFound`.
   - Update the instance's fields in-place using Pydantic's `model_validate`
     or by setting attributes directly.
   - Clear any pending update tracking for this instance in `_update_instances`.
2. **Add `refresh` to the `Session` Protocol**.

### Key design considerations

- After refresh, the instance's tracked changes should be cleared — it now
  mirrors the DB state exactly.
- Use the same `_hydrate()` pattern from `ScalarResult` to rebuild field
  values, but apply them to the existing instance rather than creating a new one.
- Refreshing an instance that is staged for deletion should still work
  (user may want to inspect before confirming).

## Acceptance Criteria

- [x] `session.refresh(instance)` reloads all fields from DB
- [x] Raises `ValueError` if instance has no `_id`
- [x] Raises `NotFound` if document was deleted from DB
- [x] Pending tracked changes for the instance are cleared after refresh
- [x] Added to `Session` Protocol
- [x] Tests in `tests/test_session_refresh.py`
