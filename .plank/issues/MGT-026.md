# MGT-026: Session state properties — `.dirty`, `.new`, `.deleted`

## Description

SA v2 exposes `session.dirty`, `session.new`, and `session.deleted` as
read-only properties to inspect the session's pending state. mongotic already
tracks this data internally (`_add_instances`, `_update_instances`,
`_delete_instances`) — expose it as a clean public API.

## Expected API

```python
with Session() as session:
    user = User(name="Alice", age=25)
    session.add(user)

    print(session.new)      # [User(name="Alice", ...)]
    print(session.dirty)    # []
    print(session.deleted)  # []

    session.flush()

    user.age = 26
    print(session.new)      # []
    print(session.dirty)    # [User(name="Alice", age=26)]

    session.delete(user)
    print(session.deleted)  # [User(name="Alice", ...)]
```

## Implementation Guidelines

1. **Add read-only properties to `_Session`**:
   - `new` → returns `list(self._add_instances)` (shallow copy).
   - `dirty` → returns list of instances in `self._update_instances` that have
     pending field changes.
   - `deleted` → returns `list(self._delete_instances)` (shallow copy).
2. **Return shallow copies** to prevent external mutation of internal state.
3. **Add properties to `Session` Protocol**.

### Key design considerations

- SA v2's `session.dirty` returns an `IdentitySet`. For simplicity, return
  a plain `list` — mongotic has no identity map.
- `dirty` should only include instances that actually have changed fields,
  not every instance ever fetched. Check `_update_instances` dict for
  instances with non-empty change sets.
- After `flush()`, `new` and `dirty` should be empty (staged ops cleared).
  `deleted` should also be empty after flush.
- These are **inspection-only** properties — mutating the returned list
  should not affect session state (hence shallow copy).

## Acceptance Criteria

- [x] `session.new` returns list of staged-for-insert instances
- [x] `session.dirty` returns list of instances with pending field changes
- [x] `session.deleted` returns list of staged-for-deletion instances
- [x] All properties return empty lists after `flush()`
- [x] Properties are read-only
- [x] Added to `Session` Protocol
- [x] Tests in `tests/test_session_state.py`
