# MGT-008: Model __indexes__ definition

> **Deferred to v0.4.0.** This feature is independent of the SA v2 alignment
> work in EPIC-001 and is lower priority for v0.3.0.

## Description

Allow model classes to declare MongoDB indexes via a `__indexes__` class attribute.
Indexes should be created automatically when a dedicated `create_indexes()` call
is made.

## Expected API

```python
from pymongo import ASCENDING, DESCENDING
from pymongo.operations import IndexModel

class User(MongoBaseModel):
    __tablename__ = "users"
    __indexes__ = [
        IndexModel([("email", ASCENDING)], unique=True),
        IndexModel([("created_at", DESCENDING)]),
    ]

    email: str
    created_at: datetime

# Apply indexes explicitly
from mongotic import create_indexes
create_indexes(engine, User)
```

## Implementation Notes

- `__indexes__` is a list of pymongo `IndexModel` objects
- Add a `create_indexes(engine, *models)` utility function
- Use `collection.create_indexes(model.__indexes__)`
