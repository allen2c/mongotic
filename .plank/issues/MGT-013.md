# MGT-013: SA v2 alignment documentation

## Description

Write documentation (via mkdocs-material) explaining mongotic's SA v2-aligned
API, including migration notes from v0.2.0, and the design rationale.

## Documents to Write

### 1. `docs/quickstart.md`
End-to-end example: define model → create engine → session → CRUD.

### 2. `docs/querying.md`
- `select()` + `.where()` + `.order_by()` + `.limit()` + `.offset()`
- `session.scalars()` → `ScalarResult`
- `.all()`, `.first()`, `.one()`, `.count()`, `.exists()`
- `session.get(Model, id)`

### 3. `docs/session.md`
- `sessionmaker` and `Session`
- `add()`, `delete()`, `flush()`, `commit()`, `rollback()`
- Context manager pattern
- Difference between `flush()` and `commit()`

### 4. `docs/migration-v0.2-to-v0.3.md`
Migration guide for breaking changes:

| v0.2.0 | v0.3.0 |
|--------|--------|
| `session.query(User).filter(...)` | `session.scalars(select(User).where(...))` |
| `.filter()` / `.filter_by()` | `.where()` |
| `.first()` raises `NotFound` | `.first()` returns `None` |
| `.all()` limited to 5 by default | no default limit |

### 5. `docs/design.md`
Why SA v2 semantics? Statement/Result separation, session lifecycle.
