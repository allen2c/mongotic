# MGT-023: Range operator — `.between()`

## Description

Add `.between(low, high)` to `ModelField` as syntactic sugar for a combined
`>=` and `<=` condition, matching SA v2's `column.between()`.

## Expected API

```python
# Equivalent to: and_(User.age >= 18, User.age <= 65)
stmt = select(User).where(User.age.between(18, 65))

# Works with dates too
stmt = select(Order).where(
    Order.created_at.between(start_date, end_date)
)
```

## Implementation Guidelines

1. **Add `between(self, low, high)` to `ModelField`**:
   - Add a `BETWEEN` variant to `Operator` enum.
   - Return a `ModelFieldOperation` with the `BETWEEN` operator and value
     `(low, high)`.
2. **Update `to_mongo_filter()`** to expand `BETWEEN` into the inline MongoDB
   filter `{"field": {"$gte": low, "$lte": high}}` — this is the optimal
   representation, allowing MongoDB to use a single index scan.
3. **No dependency on MGT-018** — this uses the existing `ModelFieldOperation`
   mechanism rather than `CompoundFilter`.

### Key design considerations

- MongoDB can use a single index scan for `{"$gte": low, "$lte": high}` on
  the same field — this is the optimal representation.
- Inclusive bounds (`>=` and `<=`) match SA v2's `between()` semantics.
- If users need exclusive bounds, they can use explicit `>` and `<` operators.

## Acceptance Criteria

- [x] `User.age.between(18, 65)` produces `{"age": {"$gte": 18, "$lte": 65}}`
- [x] Works with `datetime`, `int`, `float`, `str` types
- [x] Composes with other `.where()` conditions
- [x] Tests in `tests/test_between.py`
